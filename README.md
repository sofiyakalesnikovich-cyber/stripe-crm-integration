### 2. UML Sequence Diagram (Диаграмма последовательности)
Ниже представлена схема взаимодействия систем в процессе оплаты курса:

```mermaid
sequenceDiagram
    autonumber
    actor Студент
    participant Фронтенд как Сайт Школы
    participant Бэкенд как Сервер CRM
    participant API Stripe как Платежный Шлюз

    Студент->>Фронтенд: Нажимает "Купить курс" и вводит данные карты
    Фронтенд->>Бэкенд: Отправляет запрос на создание заказа (Order ID, Сумма)
    Бэкенд->>API Stripe: Отправляет запрос POST /v1/charges (Данные карты, Сумма)
    
    alt Успешная оплата
        API Stripe-->>Бэкенд: Возвращает статус 200 OK (Payment Intent ID, Success)
        Бэкенд->>Бэкенд: Меняет статус студента в базе на "Оплачено"
        Бэкенд-->>Фронтенд: Возвращает ответ "Успех"
        Фронтенд-->>Студент: Показывает экран "Оплата прошла успешно!"
    else Ошибка (нет денег на карте)
        API Stripe-->>Бэкенд: Возвращает статус 402 Payment Required (Error: Card Declined)
        Бэкенд-->>Фронтенд: Передает ошибку оплаты
        Фронтенд-->>Студент: Показывает сообщение "Недостаточно средств. Попробуйте другую карту"
    end
